# ADR-008: RDRA 3.0の採用

## ステータス

承認（ADR-003, ADR-004, ADR-005 に影響）

## コンテキスト

当初RDRA 2.0をベースに設計を進めていたが、RDRA 3.0が公開されており以下の変更がある:

1. **表形式定義の導入**: ダイアグラム中心からスプレッドシートベースへ移行し、資料作成工数を削減
2. **コンテキストの正式導入**: 情報・状態・条件・バリエーションをビジネス観点でグルーピングする概念
3. **条件・バリエーションのレイヤー移動**: システム外部環境からシステムレイヤーに移動
4. **LLMツール連携**: RDRAZeroOneなど、LLMと連携した要件定義の公式サポート

また、RDRA考案者の神崎善司氏がRDRAAgent v0.6（Claude Code用）を公開しており、先行実装が存在する。

## 決定

**RDRA 3.0をベースに設計する**。

## 理由

- **最新性**: 2.0ベースで設計すると公開時点で一世代遅れたスキルになる
- **コンテキスト概念の整合**: 本スキルのバーティカルスライス設計はRDRA 3.0の「コンテキスト」と自然に対応する。2.0にはこの概念が正式に存在しない
- **条件・バリエーションの配置**: 3.0ではシステムレイヤーに移動しており、バーティカルスライス内での整理が容易
- **LLM活用の親和性**: 3.0はLLMによる要件定義を考慮した設計であり、Claude Codeスキルとしての自然な拡張

## トレードオフ

- RDRA 3.0の書籍・資料はまだ2.0ほど普及していない → ユーザーが2.0の知識で利用する場合のガイダンスを分析モードに含める
- 表形式定義との互換性 → 本スキルはYAMLフロントマター形式を採用しているため、スプレッドシート形式との直接互換はない。将来拡張でインポート/エクスポートを検討
