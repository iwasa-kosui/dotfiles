# ADR-001: 統合型1スキル + 4モード構成

## ステータス

承認

## コンテキスト

RDRA分析・仕様作成・レビュー・更新の4つの機能をどう提供するか。選択肢は:

1. **4つの独立スキル**: `rdra-analysis`, `spec-writer`, `spec-reviewer`, `spec-updater`
2. **統合型1スキル + 4モード**: `rdra-sdd` スキルにモード選択を持たせる

## 決定

**統合型1スキル + 4モード**を採用する。

## 理由

- **モード間遷移の自然さ**: 分析→仕様作成→レビューは連続したワークフローであり、1スキル内でモードを切り替えるほうが対話の流れを維持できる。分析で生成したRDRA成果物のパスを仕様作成モードが暗黙的に引き継げる
- **共有コンテキスト**: 4モードはRDRA成果物を共有する。スキルが分かれると成果物パスの受け渡しが煩雑になる
- **既存パターンの踏襲**: `resume-coach`（ヒアリング/レビューの2モード）で実証済みのパターンを拡張する
- **トリガーの明確さ**: 「RDRA」「要件定義」「仕様書」のいずれのキーワードでも1つのスキルにルーティングされる

## トレードオフ

- SKILL.mdが長くなるリスクがある → 各モードの詳細を `references/` に委譲し、SKILL.md本体は500行以下に抑える
- 4モードの識別ロジックが必要 → ユーザーの発話から意図を推定し、曖昧な場合は確認する（`resume-coach` と同じアプローチ）
