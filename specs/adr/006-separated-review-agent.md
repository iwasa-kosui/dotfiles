# ADR-006: レビューエージェントの分離

## ステータス

承認

## コンテキスト

レビューモードの実行方式をどうするか。選択肢は:

1. **スキル本体でインラインレビュー**: SKILL.md内のレビュー手順に従ってメインエージェントが実行
2. **サブエージェントに委譲**: `agents/reviewer.md` を読み込んだサブエージェントがレビューを実行

## 決定

**レビューエージェントをサブエージェントとして分離する**。`agents/reviewer.md` にレビュー手順・観点・出力フォーマットを定義し、Agent toolで実行する。

## 理由

- **確認バイアスの軽減**: メインエージェントが分析・仕様作成を行った後に同じコンテキストでレビューすると、自分の成果物に対して甘くなる傾向がある。サブエージェントはフレッシュなコンテキストでレビューを開始するため、確認バイアスが構造的に軽減される
- **既存パターンの踏襲**: `skill-creator` が `agents/grader.md` / `agents/comparator.md` でサブエージェント分離を実践済み。実証されたパターンに従う
- **コンテキストウィンドウの効率化**: レビュー手順は詳細かつ長文になる（4観点 × チェック項目）。サブエージェントに委譲することでメインのSKILL.mdを圧迫しない
- **独立実行可能**: レビューモードだけを単体で使える。既存の仕様書をレビューしたい場合に分析モードを経由する必要がない

## トレードオフ

- サブエージェント起動のオーバーヘッド → レビューは頻繁に実行する操作ではないため許容範囲
- メインエージェントとのコンテキスト共有が制限される → 必要な情報（成果物パス、RDRA成果物）を明示的に渡す
