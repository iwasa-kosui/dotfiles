# ADR-004: Mermaid図種とRDRAモデルのマッピング

## ステータス

承認

## コンテキスト

RDRAの各モデル（コンテキスト図、業務フロー、UC複合図等）をMermaidでどの図種に対応させるか。MermaidはUML完全準拠ではなく、RDRA固有の記法もサポートしていないため、代替表現が必要。

## 決定

以下のマッピングを採用する:

| RDRAモデル | Mermaid図種 | 根拠 |
|-----------|-------------|------|
| システムコンテキスト | `graph TB` | アクター・外部システム・対象システムの関係を表現 |
| ビジネスコンテキスト | `graph LR` | 業務と関係者の横並び関係 |
| ビジネスユースケース | `graph TB` | UC図非対応のためフローチャートで代替 |
| 業務フロー | `sequenceDiagram` | アクター間のやりとりを時系列で表現 |
| UC複合図（ロバストネス図） | `flowchart` + `classDef` | boundary/control/entityの3分類を色分けで表現 |
| 情報モデル | `erDiagram` | エンティティ間の関係をネイティブサポート |
| 状態モデル | `stateDiagram-v2` | 状態遷移をネイティブサポート |

## 理由

- **業務フローにシーケンス図**: RDRA業務フローはアクター間のメッセージ交換を重視する。シーケンス図はこの特性と自然にマッチする。フローチャートでは「誰が」の情報が失われやすい
- **ロバストネス図にflowchart + classDef**: Mermaidにはロバストネス図のネイティブサポートがないが、`classDef` で boundary（黄）/ control（青）/ entity（緑）のスタイルを定義し、`:::boundary` 等でノードを分類することで視覚的に等価な表現が可能。[参考実装](https://kosui.me/posts/2025/05/08/011826)で実証済み
- **GitHubプレビュー互換**: 採用した全図種はGitHub Markdownでネイティブレンダリングされる

## トレードオフ

- ロバストネス図の `classDef` スタイルはMermaidのバージョンアップで変更される可能性 → テンプレートに固定値を記述し、Specでバージョン要件を明記
- シーケンス図は複雑な分岐の表現が苦手 → 分岐が多い場合は `alt` / `opt` を使い、さらに複雑な場合はフローチャートに切り替える旨をガイドに記載
